# GitHub Upload Guide for Monthly Cache Update Workflow

## 📋 Overview
This guide lists **all files and folders** needed to run the monthly cache update workflow on GitHub Actions.

## ✅ Current System Features
Your cache system already handles:
- ✅ **Partial data**: If some crops/districts fail, others still get cached
- ✅ **Latest date tracking**: Each cached file has `cached_at` timestamp
- ✅ **Same format**: All files maintain JSON format
- ✅ **Graceful failures**: Missing data is skipped, not fatal
- ✅ **Metadata tracking**: `cache_metadata.json` tracks what was updated and when

## 🔐 GitHub Secrets Required

Go to: **Repository → Settings → Secrets and variables → Actions**

Add these secrets:

1. **HF_TOKEN**
   - Your Hugging Face token (starts with `hf_`)
   - Needs **write** permission
   - Get from: https://huggingface.co/settings/tokens

2. **DATA_GOV_API_KEY**
   - data.gov.in API key for price data
   - Get from: https://data.gov.in/

## 📊 How Cache Update Handles Partial Data

### **Example: Some crops have data, some don't**

**Scenario:** Tomato data available for all districts, but Ajwan only in 20/33 districts

**What happens:**
1. ✅ Tomato: All 33 files created successfully
   ```json
   {
     "commodity": "Tomato",
     "district": "Ahmedabad",
     "year": 2026,
     "month": 3,
     "cached_at": "2026-03-05T10:30:25.123456",
     "monthly_mean_price": 2500.50,
     "days_traded": 30,
     "price_source": "API"
   }
   ```

2. ✅ Ajwan: Only 20 files created (available districts)
   - Missing districts are skipped
   - No error, continues with next

3. ✅ Final metadata shows what was cached:
   ```json
   {
     "last_updated": "2026-03-05T10:45:00.000000",
     "update_year": 2026,
     "update_month": 3,
     "commodities_cached": ["Ajwan", "Tomato", ...],
     "districts_cached": ["Ahmedabad", "Amreli", ...]
   }
   ```

### **Example: Rainfall API fails**

**What happens:**
1. Price update: Continues (843 files)
2. Rainfall update: ⚠️ Fails
3. NDVI update: ✅ Continues (33 files)
4. Workflow: ⚠️ Shows warning but completes
5. HF Upload: Uploads whatever was successful

## 🎯 Workflow Behavior

### **Automatic Monthly Run:**
- Runs on **22nd of every month at 2:00 AM UTC**
- Updates cache with latest data
- Uploads to Hugging Face automatically

### **Manual Run:**
- Go to: **Actions → Monthly Cache Update → Run workflow**
- Options:
  - **Year/Month**: Override target period
  - **Skip prices**: Rainfall + NDVI only
  - **Force deploy**: Include large model/processed files

### **What Gets Updated:**
1. **Prices**: 30-day average from data.gov.in
2. **Rainfall**: Previous month from Open-Meteo
3. **NDVI**: Previous month from local CSV

## 📝 Cache File Format

### **Individual Cache File Example:**
```json
{
  "commodity": "Tomato",
  "district": "Ahmedabad",
  "year": 2026,
  "month": 2,
  "cached_at": "2026-02-24T12:30:00.526754",
  "monthly_mean_price": 11537.599047619047,
  "days_traded": 30,
  "data_source": "CSV",
  "last_date": "2026-08-01"
}
```

### **Key Fields:**
- `cached_at`: When this file was created/updated
- `year`, `month`: Data period (not cache date)
- `last_date`: Latest date where data was found
- All dates preserved to track data freshness

## 🚀 Quick Upload Steps

### **1. Create .gitignore**
```bash
# In Crop_Price_V2 folder
echo "monthly_cache/" > .gitignore
echo "__pycache__/" >> .gitignore
echo "*.pyc" >> .gitignore
echo ".env" >> .gitignore
echo "*.log" >> .gitignore
```

### **2. Initialize Git (if not done)**
```bash
cd d:\SEM_six_SGP\Crop_Price_V2
git init
git add .
git commit -m "Initial commit: Crop Price Forecasting with monthly cache workflow"
```

### **3. Push to GitHub**
```bash
git remote add origin https://github.com/YOUR_USERNAME/Crop_Price_V2.git
git branch -M main
git push -u origin main
```

### **4. Add Secrets**
- GitHub → Settings → Secrets → Actions
- Add `HF_TOKEN` and `DATA_GOV_API_KEY`

### **5. Test Workflow**
- GitHub → Actions → Monthly Cache Update
- Click "Run workflow"
- Select options and run
